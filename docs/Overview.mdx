# Ossy Packages

Welcome to the **Ossy Ecosystem** — a collection of packages for building applications with our CMS and design system. This Storybook serves as the central documentation for all packages in the monorepo.

## Navigation

Use the sidebar to explore:

- **App** — Getting started with the connected app setup
- **Pages** — Page templates and layouts
- **Types** — TypeScript interfaces and types
- **Design System** — UI components (Layout, Feedback, Navigation, Actions, Display, Inputs, and more)

## Packages

### Application Layer

| Package | Description |
|---------|-------------|
| **@ossy/connected-components** | CMS-connected React components. Provides `App`, routing, theming, workspace context, and SDK integration. |
| **@ossy/app** | Build and dev server for Ossy apps. Page-centric: create `src/pages.jsx` — no App.jsx required. |
| **@ossy/pages** | Page templates and layouts for building content-driven interfaces. |

### UI & Design

| Package | Description |
|---------|-------------|
| **@ossy/design-system** | Core React component library. Themable, supports theme switching, SSR, and React 19. |
| **@ossy/themes** | Design system themes (e.g. `CloudLight`, `CloudDark`). |

### Data & API

| Package | Description |
|---------|-------------|
| **@ossy/sdk** | Software Development Kit for interacting with Ossy services. |
| **@ossy/sdk-react** | React bindings for the SDK — hooks and components for data fetching. |

### Routing

| Package | Description |
|---------|-------------|
| **@ossy/router** | Base router with multi-language support and localized paths. |
| **@ossy/router-react** | React router for SSR and multi-language applications. |

### Foundation

| Package | Description |
|---------|-------------|
| **@ossy/types** | Shared TypeScript types used across the ecosystem. |
