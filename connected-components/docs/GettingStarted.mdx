# Getting Started

The **Ossy Ecosystem** provides routing, theming, workspace context, and SDK integration.

## @ossy/cli vs @ossy/connected-components

- **@ossy/cli** — Use when you want the fastest path: run `npx @ossy/cli dev` with `src/pages/*.page.jsx` (or `src/pages.jsx`) and `src/config.js`. It bundles your app, runs the dev server, and handles SSR.
- **@ossy/connected-components** — Use when you have a custom setup (Next.js, Vite, Remix, etc.). Import the `App` component and wire it with your own bundler and server.

## Installation

```bash
npm install @ossy/connected-components @ossy/design-system @ossy/themes @ossy/router-react @ossy/sdk @ossy/sdk-react
```

## Setup with @ossy/cli

Create `*.page.jsx` files in `src/`:

```jsx
// src/home.page.jsx
import React from 'react'
export default () => <h1>Welcome</h1>
```

Each file becomes a route: `home.page.jsx` → `/`, `about.page.jsx` → `/about`. Optionally export `metadata` for multi-language: `export const metadata = { path: { en: '/about', sv: '/om' } }`. Add `src/config.js` for `workspaceId`, `theme`, etc. Run `npx @ossy/cli dev` or `npx @ossy/cli build`.

## App component

The `App` component wraps routing, theming, and SDK. With `@ossy/cli`, it's used internally. For custom setups, you can use it directly:

```jsx
import { App } from '@ossy/connected-components';
import { CloudLight } from '@ossy/themes';

export default () => (
  <App
    workspaceId="your-workspace-id"
    theme={CloudLight}
    pages={[
      { id: 'home', path: '/', element: <HomePage /> },
      { id: 'about', path: '/about', element: <AboutPage /> },
    ]}
  />
);
```

## Configuration

| Prop | Description |
|------|-------------|
| `workspaceId` | Your workspace identifier |
| `theme` | Theme object (e.g. `CloudLight`, `CloudDark`) |
| `themes` | Multiple themes for switching via `useTheme` |
| `pages` | Array of page configs with `id`, `path`, and `element` |
| `apiUrl` | Optional API base URL |
| `devMode` | Enables theme editor when `true` |

## Theming

Use themes from `@ossy/themes` or pass custom theme objects. For multiple themes with runtime switching, pass both `theme` (default) and `themes`:

```jsx
<App
  theme="light"
  themes={{
    light: CloudLight,
    dark: CloudDark,
  }}
  pages={pages}
/>
```
