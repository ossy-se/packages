# Getting Started

The **Ossy Ecosystem** provides routing, theming, workspace context, and SDK integration. Use either a **page-centric** setup (recommended) or the `App` component directly.

## Installation

```bash
npm install @ossy/connected-components @ossy/design-system @ossy/themes @ossy/router-react @ossy/sdk @ossy/sdk-react
```

## Page-centric (recommended)

With `@ossy/app`, create `src/pages.jsx` — no App.jsx needed:

```jsx
import React from 'react'

export default [
  { id: 'home', path: '/', element: <HomePage /> },
  { id: 'about', path: '/about', element: <AboutPage /> },
]
```

The build uses the default app wrapper with your pages. Add `src/config.js` for `workspaceId`, `theme`, etc.

## App component

For full control, use the `App` component directly:

```jsx
import { App } from '@ossy/connected-components';
import { CloudLight } from '@ossy/themes';

export default () => (
  <App
    workspaceId="your-workspace-id"
    theme={CloudLight}
    pages={[
      { id: 'home', path: '/', element: <HomePage /> },
      { id: 'about', path: '/about', element: <AboutPage /> },
    ]}
  />
);
```

## Configuration

| Prop | Description |
|------|-------------|
| `workspaceId` | Your workspace identifier |
| `theme` | Theme object (e.g. `CloudLight`, `CloudDark`) |
| `themes` | Multiple themes for switching via `useTheme` |
| `pages` | Array of page configs with `id`, `path`, and `element` |
| `routes` | Legacy: array of route configs with `path` and `element` (converted to pages) |
| `apiUrl` | Optional API base URL |
| `devMode` | Enables theme editor when `true` |

## Theming

Use themes from `@ossy/themes` or pass custom theme objects. For multiple themes with runtime switching, pass both `theme` (default) and `themes`:

```jsx
<App
  theme="light"
  themes={{
    light: CloudLight,
    dark: CloudDark,
  }}
  pages={pages}
/>
```
